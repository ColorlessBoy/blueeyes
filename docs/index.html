Public Announcement Logic in Lean 4

红眼睛 / 绿眼睛逻辑题的形式化

这个文档展示项目如何在 Lean 4 + Mathlib 中构建世界、知识、公开公告、无人离开的公共更新， 以及有限世界上的红眼人数计数，再证明经典解法中的计数归纳。

至少有一人红眼

The Puzzle

问题的逻辑骨架

1

观察限制

每个人能看见其他人的眼睛颜色，但看不见自己的颜色。

2

公开公告

公告“至少有一个人是红眼睛”，把零红眼的世界一次性排除。

3

无人离开

每个安静夜晚都会成为新的公共信息，继续排除更小的红眼人数。

4

归纳结论

若实际有 r 个红眼者，他们会在第 r 晚离开。

Reasoning

公告为什么会改变局面

没有公告之前

每个人都知道其他人的颜色，但不知道自己的颜色。如果某人看到 seen 个红眼者，他面对的候选世界只有两个：总数是 seen，或者总数是 seen + 1。

公告的作用

公告“至少有一个红眼者”不会告诉大家谁是红眼者，但它把 totalRed = 0 的世界变成公共不可能。这是单红眼情形能够启动的原因。

安静夜晚的作用

如果第一晚没人离开，就说明没有人能在公告后立即推出自己是红眼者，于是 totalRed = 1 也被排除。之后每个安静夜晚都把下一个较小总数排除。

1 0 0 已由公告排除 第 1 晚
2 1 0, 1 第 2 晚
r r - 1 0, 1, ..., r - 1 第 r 晚

Epistemic Semantics

认识逻辑层如何表达故事

A

世界

一个世界是完整颜色赋值：每个参与者都有红或绿。Lean 中用 World Agent 表示，其中 Agent 可以是任意参与者类型。

B

不可区分

对参与者 i 来说，两个世界不可区分，当且仅当除 i 之外所有人的颜色一致。这正是“看不到自己”的形式化。

C

知识

Knows M i w P 表示在公共模型 M 中，所有 i 无法从 w 区分的可能世界都满足 P。

D

公共更新

公告和无人离开都被建模成对可能世界集合的收缩。每一次收缩都会成为下一轮知识判断的背景。

公告后 afterGuruAnnouncement w ↔ AtLeastOneRed w
一晚安静后 afterOneQuietNight M w ↔ M w ∧ ∀ i, ¬ WouldLeave M i w
连续安静夜晚 afterQuietNights (n + 1) = afterOneQuietNight (afterQuietNights n)

关键区分：人人知道 ≠ 公共知识

两个人都红时，每个人都能看见另一个红眼者，所以每个人都知道“至少有一个红眼者”。 但这仍不等于公共知识。岛民 0 仍认为自己可能是绿眼；在那个可能世界里， 岛民 1 又会认为全绿世界可能。因此二阶互知已经断掉。 

theorem allRedTwo_everyoneKnows_atLeastOneRed :
    EveryoneKnows initialModel allRedTwo AtLeastOneRed

theorem allRedTwo_not_commonKnowledge_atLeastOneRed :
    ¬ CommonKnowledge initialModel AtLeastOneRed allRedTwo

theorem allRedTwo_not_commonKnowledge_everyoneKnows_atLeastOneRed :
    ¬ CommonKnowledge initialModel
      (fun w => EveryoneKnows initialModel w AtLeastOneRed)
      allRedTwo

五人全红版本

五个人全是红眼时也是一样：每个人都知道至少有一个红眼者，但这件事仍然不是公共知识。 Lean 中的证明沿着下面这条不可区分链走到全绿世界： 

RRRRR ~ GRRRR ~ GGRRR ~ GGGRR ~ GGGGR ~ GGGGG

theorem allRedFive_everyoneKnows_atLeastOneRed :
    EveryoneKnows initialModel allRedFive AtLeastOneRed

theorem allRedFive_not_commonKnowledge_atLeastOneRed :
    ¬ CommonKnowledge initialModel AtLeastOneRed allRedFive

theorem allRedFive_not_commonKnowledge_everyoneKnows_atLeastOneRed :
    ¬ CommonKnowledge initialModel
      (fun w => EveryoneKnows initialModel w AtLeastOneRed)
      allRedFive

公共知识固定点

文件中将公共知识定义为所有有限层级的互知： P、人人知道 P、人人知道人人知道 P，依此类推。 定理 commonKnowledge_unfold 给出固定点展开。 

def CommonKnowledge (M : Model Agent) (P : Propn Agent) : Propn Agent :=
  fun w => ∀ n, IteratedEveryoneKnows M P n w

theorem commonKnowledge_unfold :
    CommonKnowledge M P w ↔
      P w ∧ EveryoneKnows M w (CommonKnowledge M P)

离开报告和状态转移

NightState 同时记录当前公共模型和仍在场的人。夜晚结束时，观察到的离开名单也会成为公共信息， 因此模型会保留那些预测出同一离开名单的世界。 

structure NightState (Agent : Type u) where
  model : Model Agent
  present : Agent → Prop

def nextNightState
    (S : NightState Agent) (actual : World Agent) :
    NightState Agent

Formal Model

从故事到 Lean 命题

世界与观察

一个世界给每个参与者分配眼睛颜色。两个世界对参与者 i 不可区分，意味着除 i 自己之外，所有人的颜色都相同。 

structure World (Agent : Type u) where
  color : Agent → EyeColor

def indistinguishable
    (i : Agent) (w v : World Agent) : Prop :=
  ∀ j, j ≠ i → v.color j = w.color j

知识与公开更新

Knows 是标准 Kripke 风格定义：所有仍可能且不可区分的世界都满足 P。公开公告则把模型限制到满足公告命题的世界。 

def Knows (M : Model Agent)
    (i : Agent) (w : World Agent) (P : Propn Agent) : Prop :=
  ∀ v, M v → indistinguishable i w v → P v

def publicAnnouncement
    (M : Model Agent) (A : Propn Agent) : Model Agent :=
  fun w => M w ∧ A w

无人离开的夜晚

若没有人知道自己是红眼者，就没有人离开。这个事实公开后，模型会再次收缩。 

def QuietNight (M : Model Agent) : Propn Agent :=
  fun w => ∀ i, ¬ WouldLeave M i w

def afterOneQuietNight (M : Model Agent) : Model Agent :=
  publicAnnouncement M (QuietNight M)

观察相容性

某人看到 seen 个红眼者时，总红眼人数只有两种可能： 自己不是红眼者，所以总数是 seen；或者自己是红眼者，所以总数是 seen + 1。 

def compatibleWithObservation
    (seen totalRed : Nat) : Prop :=
  totalRed = seen ∨ totalRed = seen + 1

安静夜晚后的可能世界

公告排除 totalRed = 0。连续 quietNights 晚无人离开后，还会排除所有 totalRed ≤ quietNights 的世界。 

def possibleAfterQuietNights
    (quietNights seen totalRed : Nat) : Prop :=
  compatibleWithObservation seen totalRed ∧
    0 < totalRed ∧ quietNights < totalRed

有限世界中的真实计数

Mathlib 的 Fintype 和 Finset 用来表达有限参与者集合。 totalRed 是真实红眼总数，seenRed 是从红眼集合中移除自己后的数量。 

def redAgents (w : World Agent) : Finset Agent :=
  Finset.univ.filter fun i => w.color i = EyeColor.red

def totalRed (w : World Agent) : Nat :=
  (redAgents w).card

def seenRed (i : Agent) (w : World Agent) : Nat :=
  ((redAgents w).erase i).card

Proof Roadmap

从 Kripke 语义到计数归纳

01

先固定观察者

对任意参与者 i 和世界 w，定义 seenRed i w 与 totalRed w。

02

证明观察只留下两个总数

compatibleWithConcreteObservation 证明真实总数只能等于所见数或所见数加一。

03

把安静夜晚变成下界

afterQuietNights_count_lower_bound 直接从 Kripke 更新推出 quietNights < totalRed。

04

排除“自己不是红眼者”

红眼者看到 n 个红眼者，经过 n 个安静夜晚后， totalRed = n 已不可能，只剩 n + 1。

关键归纳的证明思路

假设第 n + 1 次安静更新后还有一个世界 w，但 totalRed w ≤ n + 1。归纳假设已经给出 n < totalRed w， 因此只能是 totalRed w = n + 1。取一个红眼者 i。 如果某个与 w 对 i 不可区分的剩余世界把 i 设为绿眼，那么该世界的总红眼数就是 n，违反归纳假设。 所以 i 会知道自己是红眼者并离开，和“安静夜晚”矛盾。 

theorem afterQuietNights_count_lower_bound :
    ∀ n {w : World Agent},
      afterQuietNights n w → n < totalRed w

theorem possibleAfterQuietNights_of_afterQuietNights
    (hw : afterQuietNights n w) :
    possibleAfterQuietNights n (seenRed i w) (totalRed w)

Lean 4 Details

关键定理怎样对应直觉推理

公开公告和安静夜晚的语义

公告后的模型恰好保留至少有一个红眼者的世界；一次安静夜晚更新保留仍可能且无人会离开的世界。 

theorem afterGuruAnnouncement_iff :
    afterGuruAnnouncement w ↔ AtLeastOneRed w

theorem afterOneQuietNight_iff :
    afterOneQuietNight M w ↔ M w ∧ ∀ i, ¬ WouldLeave M i w

为什么看不见自己会阻止知识

如果把自己的眼睛改成绿色后的世界仍可能，那么这个人不能知道自己是红眼者。 

theorem not_knows_red_if_green_variant_possible
    (hgreen : M (recolorSelf i EyeColor.green w)) :
    ¬ Knows M i w (IsRed i)

公共知识和状态转移

公共知识以固定点方式展开；显式离开状态转移在空离开报告时退化为 quiet-night 更新。 

theorem commonKnowledge_unfold :
    CommonKnowledge M P w ↔
      P w ∧ EveryoneKnows M w (CommonKnowledge M P)

theorem updateModelBy_empty_report_eq_quiet :
    updateModelByReport M (fun _ => True) (fun _ => False) =
      afterOneQuietNight M

具体世界接入计数证明

具体有限世界会导出计数相容性；不可区分世界保持看到的红眼人数不变。 这把 Kripke 语义层和经典计数证明连接起来。 

theorem seenRed_eq_of_indistinguishable
    (hobs : indistinguishable i w v) :
    seenRed i v = seenRed i w

theorem compatibleWithConcreteObservation
    (i : Agent) (w : World Agent) :
    compatibleWithObservation (seenRed i w) (totalRed w)

红眼者在决定性夜晚知道自己是红眼者

若实际有 r > 0 个红眼者，则红眼者看到 r - 1 个红眼者。经过 r - 1 个安静夜晚后， “自己不是红眼者”的分支已经被排除。 

theorem redEyedPerson_knows_after_quietNights
    (r totalRed : Nat) (hr : 0 < r) :
    possibleAfterQuietNights (r - 1) (r - 1) totalRed →
      totalRed = r

绿眼者仍有两个可能世界

绿眼者看到 r 个红眼者，在决定性夜晚前仍不能区分 r 和 r + 1，所以不会和红眼者同晚离开。 

theorem greenEyedPerson_still_has_two_possibilities
    (r : Nat) (hr : 0 < r) :
    possibleAfterQuietNights (r - 1) r r ∧
      possibleAfterQuietNights (r - 1) r (r + 1)

Kripke 更新推出计数下界

连续 n 个安静夜晚后仍可能的具体世界，必然有超过 n 个红眼者。这已经不再是计数层的外部假设。 

theorem afterQuietNights_count_lower_bound :
    ∀ n {w : World Agent},
      afterQuietNights n w → n < totalRed w

theorem not_afterQuietNights_of_totalRed_le
    (hle : totalRed w ≤ n) :
    ¬ afterQuietNights n w

所有红眼者在同一晚离开

决定性夜晚是 totalRed w - 1 个安静夜晚之后。此时每个红眼者都会知道自己是红眼者。 

theorem allRedAgents_wouldLeave_on_decisiveNight
    (hw : afterQuietNights (totalRed w - 1) w) :
    ∀ i, IsRed i w →
      WouldLeave (afterQuietNights (totalRed w - 1)) i w

具体有限世界中的决定性结论

对红眼者而言，经过自己看到的红眼人数那么多个安静夜晚后，任何仍相容的候选总数都必须等于真实总数。 

theorem redEyedPerson_knows_concrete_total
    (hred : IsRed i w)
    (hpossible :
      possibleAfterQuietNights (seenRed i w) (seenRed i w)
        candidateTotal) :
    candidateTotal = totalRed w

公共信息归纳的可复用形式

quietNights 个安静夜晚排除所有不超过 quietNights 的红眼人数。这是整个推理的压缩表达。 

theorem eliminated_by_quietNights
    (hle : totalRed ≤ quietNights) :
    ¬ possibleAfterQuietNights quietNights seen totalRed

Concrete Example

三人岛民：两红一绿

具体世界

这里不再使用抽象参与者，而是固定 Fin 3。岛民 0 和 1 是红眼者，岛民 2 是绿眼者。 

def twoRedOneGreen : World (Fin 3) where
  color
    | 0 => EyeColor.red
    | 1 => EyeColor.red
    | 2 => EyeColor.green

Lean 已证明的故事结论

这个世界中总共有两个红眼者。第一晚无人离开；第二晚，两个红眼者一起离开。 

theorem twoRedOneGreen_after_one_quiet_night :
    afterQuietNights 1 twoRedOneGreen

theorem twoRedOneGreen_reds_leave_together :
    WouldLeave (afterQuietNights 1) 0 twoRedOneGreen ∧
      WouldLeave (afterQuietNights 1) 1 twoRedOneGreen
